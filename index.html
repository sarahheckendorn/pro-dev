Why I Love Coding:

Code covers just about every side of me that I like to tap into: creative, problem solving, and mundane. Some of that may sound contradictory, and so it is... so I will explain

Problem solving comes with the process. Whether it's front end or back end, you have to think through how you're going to make your website work. OOP definitely has a ton of problem solving- those languages that are more than static HTML most certainly rely on you using your brain to figure out why it's working or not, what it needs to be doing, etc.

Creativity comes through in the whole big plan. Figuring out how you want the whole thing to work together- but most especially in the front end. Making it look pretty on the user side. Color schemes, flow, etc. Those are all a beautiful, creative outlet. Plus! It still pulls from the problem solving (because when is code never not problem solving?!)

The Mundane part is probably the funniest... mainly because it seems odd that a person wants to have something mundane. However, when you're working so hard with problem solving, or staying up late with the creativity side, you really do need a bit of 'so basic I can do it in my sleep' type work to help balance out the other two so you don't get burnt out. That's where mundane comes in. HTML is what I'd call the mundane. It's incredibly simple, easy to type, and again, mundane. You don't have to think a whole lot about it.

So, in summary: Coding is something that allows me to tap into all sides of my brain. I am never bored with it and it keeps my brain active. The result of your work gives a huge amount of pride. To see something you've worked so hard on come to life is so satisfactory. AND! I don't have to stick with only one project. It allows me to continue using skills on multiple projects/websites- which I think I love the most.
